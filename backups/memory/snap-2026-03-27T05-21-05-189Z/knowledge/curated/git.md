Git controla versoes de codigo.

Pontos chave:
- Commits pequenos e mensagens claras.
- Branches para features; PR para revisao.
- Merge vs rebase: escolha conforme o fluxo do time.
- .gitignore para evitar segredos e arquivos gerados.
